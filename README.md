# مدیریت تنخواه (tankhah)

وب‌اپ موبایلی فارسی (RTL) برای مدیریت تنخواه‌گردان شرکت: ثبت هزینه/درآمد، آپلود عکس فاکتور، مدیریت بدهی همکاران، داشبورد و خروجی اکسل. قابل نصب روی موبایل به‌صورت **PWA**.

> طراحی از Google Stitch گرفته شده و دست‌نخورده باقی مانده؛ فقط اتصال دیتابیس (Supabase) و منطق اپ اضافه شده است.

**🌐 نسخه‌ی زنده:** https://knightmare97.github.io/tankhah-app/ — ابتدا باید وارد شوید (ایمیل/رمز). روی موبایل از منوی مرورگر «افزودن به صفحه‌ی اصلی» را بزنید تا مثل اپ نصب شود.

## ساختار پروژه

```
app/                 ← خود اپ (همین پوشه روی هاست می‌رود)
  index.html         داشبورد
  transactions.html  لیست تراکنش‌ها + فیلتر/جستجو
  add.html           ثبت تراکنش + آپلود عکس فاکتور
  contacts.html      بدهی‌ها و همکاران
  settings.html      خروجی اکسل + پاکسازی
  config.js          آدرس و کلید عمومی Supabase
  css/app.css        استایل مشترک (فونت Vazirmatn و…)
  js/                منطق هر صفحه + توابع مشترک + تقویم شمسی
  manifest.json, sw.js, icons/   فایل‌های PWA
stitch_raw/          طراحی خام اصلی Stitch (مرجع — لازم نیست روی هاست برود)
```

## دیتابیس (Supabase)

پروژه: `tankhah` — منطقه‌ی `eu-central-1`.

- جدول `transactions`: تراکنش‌ها (مبلغ به تومان، نوع income(شارژ)/expense، وضعیت، مسئول پرداخت، `card_id`، عکس فاکتور)
- جدول `contacts`: همکاران (بدهی هر همکار از روی تراکنش‌های طلب محاسبه می‌شود — ستون ذخیره‌شده ندارد)
- جدول `cards`: کارت‌ها/کیف‌پول‌های تنخواه (موجودی هر کارت محاسبه‌ای است: شارژ − هزینه)
- باکت Storage به نام `invoices`: عکس فاکتورها

🔒 **امنیت:** ورود کاربری با Supabase Auth فعال است و RLS فقط به کاربران واردشده اجازه‌ی دسترسی می‌دهد (فضای کاری مشترک تیم). برای افزودن همکار جدید، در داشبورد Supabase → Authentication → Users یک کاربر بسازید (ثبت‌نام خودکار از داخل اپ غیرفعال است).

نکته‌ی باقی‌مانده برای سخت‌گیری بیشتر: باکت `invoices` فعلاً public است (لینک عکس‌ها با نام تصادفی). در صورت نیاز می‌توان آن را private و با Signed URL کرد.

## تاریخ‌ها

همه‌ی تاریخ‌ها **شمسی** نمایش داده می‌شوند و تقویم انتخاب تاریخ هم شمسی است (ماژول سبک `js/jdate.js` + `js/jalali-picker.js`). در دیتابیس برای مرتب‌سازی و کوئری، تاریخ به‌صورت میلادی ISO ذخیره می‌شود.

## اجرای محلی

```bash
cd app
python3 -m http.server 8123
# سپس http://localhost:8123 را باز کنید
```

(چون از Service Worker و آپلود استفاده می‌شود، باید از طریق سرور باز شود، نه فایل مستقیم.)

## انتشار (Deploy)

پوشه‌ی `app/` کاملاً استاتیک است و روی هر هاست استاتیکی کار می‌کند: Netlify، Vercel، Cloudflare Pages یا GitHub Pages. کافی است محتوای `app/` منتشر شود.

## کارهای آینده (پیشنهادی)
- افزودن ورود کاربری و محدودکردن RLS
- جداکردن Tailwind از CDN و کامپایل آن برای پروداکشن (حذف هشدار CDN)
- صفحه‌ی جزئیات تک‌تراکنش و پروفایل تک‌همکار (طرح‌شان در `stitch_raw` هست)
